description: Scaffold a new PrestaShop 9 module with the modern Symfony layout (PSR-4 src/, services.yml, routes.yml, translations). Use when the user wants to create a brand-new module from scratch.
---

## Requirements

Ask the user:
* Module technical name in `lower_snake_case`. The folder, the main PHP class (in PascalCase), the `$this->name` property, the PSR-4 namespace and the translation domain prefix all derive from it. Example: `mymodule` → class `Mymodule`, namespace `MyVendor\Mymodule\`, domain `Modules.Mymodule.*`.
* Display name shown in the Back Office (`$this->displayName`).
* Author / vendor.
* Initial version (e.g. `1.0.0`).
* Target PrestaShop version range (`ps_versions_compliancy`). For new modules default `min: 9.0.0`, `max: _PS_VERSION_`.
* Whether the module needs a configuration page, hooks, front controller(s), or admin controller(s) - those are scaffolded by the dedicated skills (`module-add-config-page-modern`, `module-add-front-controller`, `module-add-admin-controller-modern`).

## Steps

1. Create the folder `<modulename>/` at the PrestaShop root under `modules/`. Inside it create the following layout (only the files you actually need; do not pad):

    ```
    <modulename>/
    ├── <modulename>.php             # main module class (required)
    ├── composer.json                # PSR-4 autoload (required as soon as you have src/)
    ├── config/
    │   ├── services.yml             # Symfony DI (required for any modern code)
    │   └── routes.yml               # module routes (required for admin/symfony controllers)
    ├── src/                         # PSR-4 PHP code (Controllers, Form, Repository, ...)
    ├── views/
    │   ├── templates/
    │   │   ├── admin/               # Twig for modern admin controllers (.html.twig)
    │   │   ├── front/               # Smarty for ModuleFrontController (.tpl)
    │   │   └── hook/                # Smarty partials rendered by hooks (.tpl)
    │   ├── css/
    │   └── js/
    ├── translations/                # XLIFF, generated by bin/console (see module-add-translations-new)
    └── logo.png                     # 32x32 PNG, mandatory
    ```

2. Write `<modulename>.php`. Required class structure:

    ```php
    <?php
    if (!defined('_PS_VERSION_')) { exit; }

    class Mymodule extends Module
    {
        public function __construct()
        {
            $this->name = 'mymodule';
            $this->tab = 'administration';
            $this->version = '1.0.0';
            $this->author = 'Acme';
            $this->need_instance = 0;
            $this->bootstrap = true;
            $this->ps_versions_compliancy = ['min' => '9.0.0', 'max' => _PS_VERSION_];
            parent::__construct();
            $this->displayName = $this->trans('My Module', [], 'Modules.Mymodule.Admin');
            $this->description = $this->trans('What it does.', [], 'Modules.Mymodule.Admin');
        }

        public function install(): bool
        {
            return parent::install()
                && $this->registerHook(['actionFrontControllerSetMedia']);
        }

        public function uninstall(): bool { return parent::uninstall(); }
    }
    ```

3. Write `composer.json` with the PSR-4 autoload pointing at `src/`. Do NOT require `prestashop/prestashop` - the core is provided. Only require third-party libraries the module actually uses.

    ```json
    { "name": "acme/mymodule", "type": "prestashop-module",
      "autoload": { "psr-4": { "MyVendor\\Mymodule\\": "src/" } },
      "config": { "preferred-install": "dist" } }
    ```

4. Write `config/services.yml` with the standard skeleton. Do NOT register services one-by-one: use a `resource:` glob.

    ```yaml
    services:
      _defaults:
        autowire: true
        autoconfigure: true
        public: false

      MyVendor\Mymodule\:
        resource: '../src/*'
        exclude: '../src/{Entity,DTO}'
    ```

5. Create an empty `config/routes.yml` (PrestaShop auto-loads it from every enabled module). Routes are declared by `module-add-symfony-route`.

6. Add a `logo.png` (32x32). The module is now installable from Back Office > Modules > Module Manager > Upload, or via `php bin/console prestashop:module install <modulename>`.

## Do

- Use `$this->trans('...', [], 'Modules.Mymodule.<area>')` for every user-facing string. See `module-add-translations-new`.
- Inject `PrestaShopBundle\Service\Hook\HookDispatcherInterface` into your services to dispatch hooks.
- Declare every service in `config/services.yml` and let autowiring resolve constructor arguments.
- Keep the main module class thin: it is a registration point, not a place for business logic.

## Don't

- Don't use `$this->l('...')` anywhere. The legacy translator helper is gone in PS 9 modern code; only the `*-legacy` skills are allowed to mention it.
- Don't call `Hook::exec(...)` from new code. Use `HookDispatcherInterface::dispatchWithParameters()` instead.
- Don't put PHP files outside `src/` once you have a namespace; PrestaShop will not autoload them.
- Don't `require` PrestaShop core packages in `composer.json`; they are provided at runtime.
